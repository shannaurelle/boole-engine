Bitstring for Boole-Mobius Transform

Forming the Bitstring from Truth Tables

The bitstring for a truth table must have its first chracter as the first result from the first row, second character as the second result from the second row, and so on.

pq result
00 0
01 1
10 0
11 1

From this example table above, the bitstring is 0101 because the value from the first row on the result column is 0, second row is 1, third row is 0, and the fourth row is 1.

Forming the Bitstring from an Expression

Get the encoding table of the simplified expression by inserting the expression in the Boole Engine and click submit. Then, to form the bitstring from the encoding, write the first bit of the bitstring as 1 if the first term in the encoding table is present in the simplified expression, and it is 0 otherwise. For the second bit of bitstring, it is equal to 1 if the second term in the encoding table is present in the simplified expression and 0 if otherwise.

Let us say we have the expression

p+q

Its encoding table is given below

Term Encoding Present(1)/Absent(0)
1 00 0
p 01 1
q 10 1
pq 11 0

Reading from the third column, the bitstring is 0110 because the value in the first row is 0, the value in the second row is 1, the value in the third row is 1, and the value in the fourth row is 0.

Transforming Truth Table bistring to ANF bitstring and vice versa

This is done by creating the Boole-Mobius transform matrix and multiplying the bitstring as a vector to the Boole-Mobius transform matrix. In the source code upon inspection, the Boole-Mobius transform matrix is created as a 2n by 2n matrix where n is the number of variables and the values are populated with this simple rule: if the logical AND of row i and column j is equal to i, then the value at row i and column j is 1, and 0 otherwise.
